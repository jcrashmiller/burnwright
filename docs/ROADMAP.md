# Roadmap

This is a living document tracking planned features, known gaps, and ideas under consideration. Items are loosely grouped by priority and scope, not by timeline.

---

## In progress

### GUI (burnwright_gui.py)
A PySide6 desktop interface that wraps the same CLI pipeline. Single-window form layout — format, name, inputs, output, options — with a live log area and dual progress bars (current step + overall job). The CLI remains fully functional independently; the GUI is a front end, not a replacement.

---

## Near term

### Smarter disc manifest
The current manifest lists raw segment filenames. It should:
- Extract episode titles or numbers from source filenames where possible
- Flag any episode that was split across discs (should not happen in episode mode, but worth noting if it does)
- For movies, note the split timestamps and which disc covers which portion of the runtime
- Format cleanly enough to print and tuck into a jewel case

### Auto-generated disc label / insert
A simple generated image — show title, disc number, episode list or runtime range — sized for a standard CD jewel case insert. No design skills required from the user. Plain text on a background, generated by ImageMagick or Pillow. Optional.

### Basic auto-menu for episode discs
When burning a multi-episode disc, optionally generate a simple static menu image showing episode numbers and titles (or just numbers if titles are unavailable), encode it as a VCD/SVCD still sequence, and wire it into the PBC. The menu image is auto-generated — thumbnail grab from each episode at a configurable timestamp, episode number label, clean layout. No GIMP required.

Movies do not get menus by default. A movie disc should start playing immediately, like putting in a tape. Menus for movies are opt-in only.

### Burning integration
A user config section (`disc_maker_config.json` or a separate `burnwright_user.json`) that stores:
- Preferred burning application (K3b, Brasero, cdrdao direct)
- Path to burner executable
- Target optical drive device
- Auto-launch preference (offer vs. launch automatically)

When a job completes, the GUI offers a **Burn these discs** button that launches the configured burner with the output queue pre-loaded. K3b and Brasero both support command-line project loading. This closes the loop from source file to physical disc without requiring the user to manually locate and load files in a separate application.

Batch burning support: queue all discs in sequence, prompt for disc swap between burns.

## Medium term

### Progress feedback improvements
- ETA based on actual encode speed rather than linear estimate
- Per-disc time remaining in the overall progress bar
- Estimated total job time shown at the start before encoding begins

### Subtitle support
- Detect subtitle tracks in source MKV files
- Offer to burn subtitles into the video stream (hardcode) via ffmpeg `subtitles` filter
- Language selection when multiple subtitle tracks are present
- Configurable font, size, and position

### Source validation improvements
- Pre-encode check: verify source file is not corrupted before starting a long job
- Warn if source audio has sync issues detectable before encode
- Warn if source has variable framerate (VFR) that may cause issues

### Disc image verification
- After vcdxbuild, mount the image via CDEmu automatically and verify the MPEG streams are readable
- Optionally decode the first and last few seconds of each stream to confirm no corruption at split points

## Longer term / under consideration

### BundleForge integration
burnwright as an AppPack component — installable as part of a curated media tools bundle alongside Handbrake, MakeMKV, and similar tools.

### CVD support
CVD (China Video Disc) uses MPEG-2 at VCD resolution. vcdimager support is limited and player support outside Asian markets is minimal. Low priority but architecturally trivial to add — it is a config entry and a different mplex format flag.

### DVD-Video output
`dvdauthor` handles DVD-Video authoring with proper menu support. A `--format dvd` option would dramatically expand compatibility — DVD ISOs play in Kodi, VLC, and on every standalone player made since 1997. The quality ceiling is also significantly higher than SVCD. This is a larger scope addition but fits naturally in the same pipeline.

### Batocera / EmulationStation integration
burnwright running on a Linux-based all-in-one gaming and media console. VCD/SVCD disc detection and auto-launch via a disc-type detection script. Drop any disc in — game launches the right emulator, VCD launches mplayer, DVD launches VLC, audio CD launches a music player. The disc type determines the handler automatically.

### Web UI (Flask)
An optional browser-based interface for headless or remote operation. Useful for running burnwright on a server or NAS and submitting jobs from another machine. Lower priority than the desktop GUI since the primary use case is a local workstation.

## Known gaps and bugs

- `--scene-detect` scans only the target file — if multiple files in a job require splitting (unusual but possible), each is scanned correctly. No known issues here as of current version.
- Progress bar uses `\r` terminal control and displays correctly in standard terminals. May not render correctly in all terminal emulators or when output is piped.
- vcdxbuild DTD validation produces warnings for non-standard stream parameters (SVCD with 480×272 resolution). These are harmless and the disc images play correctly, but the warnings are noisy. A future version may suppress them or pass a `--no-validate` flag.
- Capacity calculation uses theoretical bitrate math rather than actual encoded file sizes. Real encode sizes vary with content complexity. The 95% safety margin handles most cases but very high-motion content at maximum bitrate could exceed disc capacity. Post-encode size verification and warning is planned.

## Philosophy note

Features are added when they make the tool more useful for its actual purpose: getting your media onto physical discs you own, in a format you control, without unnecessary friction or dependency. Features that would make burnwright phone home, require an account, limit your choices, or insert themselves between you and your media will not be added.

The roadmap is a wish list, not a commitment. This is a personal tool that happens to be open source. If something here is useful to you, great. If you want to contribute something, that is welcome. If you want to fork it and take it somewhere else entirely, that is exactly what open source is for.
